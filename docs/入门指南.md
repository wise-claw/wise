# 快速入门

> 从安装到首次 WISE 会话的快速上手指南。

如果你是WISE的新用户，请按以下顺序完成各步骤。

1. [安装](#安装) — 安装 WISE 插件并运行初始 setup
2. [首次会话](#首次会话) — 用 autopilot 运行你的第一个任务
3. [配置](#配置) — 按项目自定义设置与 agent 模型

### 本指南涵盖内容

- 如何安装 WISE 插件
- 运行首次 autopilot 会话并理解其流程
- 配置用户级与项目级设置

### 前置条件

- 必须已安装 [Claude Code](https://docs.anthropic.com/claude-code)
- 需要 Claude Max/Pro 订阅，或 Anthropic API key

---

## 安装

WISE 提供两种使用面，且设计上可并存：

| 使用面 | 你能获得什么 | 推荐安装方式 |
|---|---|---|
| **Claude Code 插件**（`wise@wise`） | 会话内 skills、agents、hooks、statusline、MCP 服务器 — `/autopilot`、`/ralph`、`/ultrawork`、`/team` 等斜杠命令 | Marketplace 插件安装（见下方步骤 1–2） |
| **终端 CLI**（`wise` 二进制，包名 `wise`） | Shell 命令：`wise setup`、`wise update`、`wise team`、`wise ask`，以及已硬弃用的 `wise autoresearch` 兼容入口 | `npm i -g wise@latest` |

大多数用户需要**两者兼有**：插件负责会话内体验，npm CLI 负责 shell 侧自动化与更新。并行使用完全支持 — `wise update` 与 `wise setup` 是幂等的，会检测插件安装以避免重复注册会话内 skills。

### 步骤 1：添加 marketplace 源

在 Claude Code 内运行：

```bash
/plugin marketplace add https://github.com/wise-claw/wise
```

### 步骤 2：安装插件

添加 marketplace 后，安装插件：

```bash
/plugin install wise
```

### 步骤 2b（可选但推荐）：安装终端 CLI

若希望在 shell 中使用 `wise setup`、`wise update`、`wise team`、`wise ask` 等：

```bash
npm i -g wise@latest
```

两者可同时安装。CLI 会自动检测插件安装，不会在 `~/.claude/skills/` 下重复注册 skills（若你曾遇到重复 skill 问题

### 步骤 3：运行初始 setup

安装完成后，在 Claude Code 中输入以下任一方式：

```bash
# 方式 1：自然语言
setup wise

# 方式 2：skill 命令
/wise:wise-setup
```

### 前置条件摘要

| 项目 | 要求 |
|------|------|
| Claude Code | 必须已安装 |
| 认证 | Claude Max/Pro 订阅，或 `ANTHROPIC_API_KEY` 环境变量 |

### 选择 setup 范围

#### 项目级 setup（推荐）

仅对当前项目启用 WISE：

```bash
/wise:wise-setup --local
```

- 设置保存到 `./.claude/CLAUDE.md`
- 不影响其他项目
- 保留现有全局 `CLAUDE.md`

#### 全局 setup

对所有 Claude Code 会话启用 WISE：

```bash
/wise:wise-setup
```

- 设置保存到 `~/.claude/CLAUDE.md`
- 应用于所有项目

> ⚠️ **警告：** 全局 setup 现在会在修改你的基础 `~/.claude/CLAUDE.md` 前明确询问。默认选项仍是覆盖。若选择保留模式，普通 `claude` 将保持你的基础配置，`wise` 会强制加载 WISE 配套配置。

### 验证安装

确认一切正常，运行诊断工具：

```bash
/wise:wise-doctor
```

它会检查：

- 依赖安装状态
- 配置文件错误
- Hook 安装状态
- Agent 可用性
- Skill 注册状态

### 从本地 checkout 运行

若你在开发 WISE，或想从特定分支测试未发布功能，可将本地 checkout 作为插件启动 Claude Code：

```bash
wise --plugin-dir /path/to/wise setup --plugin-dir-mode
```

这会直接从 checkout 加载 agents、skills 和 commands，而无需复制到 `~/.claude/`。详细说明与其他流程见 [本地插件安装.md](./本地插件安装.md)。插件目录标志与模式的完整决策矩阵见 [REFERENCE.md 中的 Plugin directory flags 章节](./REFERENCE.md#plugin-directory-flags)。

### 平台支持

| 平台 | 安装方式 | Hook 类型 |
|----------|--------------|-----------|
| macOS | Claude Code Plugin | Bash (.sh) |
| Linux | Claude Code Plugin | Bash (.sh) |
| Windows | 推荐 WSL2 | Node.js (.mjs) |

> ℹ️ **说明：** 原生 Windows 支持为实验性。WISE 依赖 tmux，原生 Windows 不可用。请改用 WSL2。

### 更新

WISE 每 24 小时自动检查更新。手动更新请重新运行插件安装命令。

> ⚠️ **警告：** 插件更新后，请再次运行 `/wise:wise-setup` 以应用最新配置。

### 卸载

```bash
/plugin uninstall wise@wise
```

## 首次会话

WISE 安装完成后，可立即运行第一个任务。打开 Claude Code 并输入：

```bash
autopilot build me a hello world app
```

这一行就足够让 WISE 自动运行完整开发流水线。

### 会发生什么

当 WISE 检测到 `autopilot` 魔法关键词时，会启动 5 阶段流水线：

### 阶段 1：Expansion（扩展）

`analyst` 与 `architect` agent 分析想法、澄清需求并产出技术规格。

### 阶段 2：Planning（规划）

`planner` agent 创建执行计划。`critic` agent 审查计划并识别缺口。

### 阶段 3：Execution（执行）

`executor` agent 编写代码。需要时多个 agent 并行工作。

### 阶段 4：QA（质量保证）

验证构建成功且测试通过。自动修复失败并重新验证。

### 阶段 5：Validation（验收）

专家 agent 对功能、安全与代码质量做最终审查。全部通过后工作完成。

### HUD 状态显示

工作进行中，可在 Claude Code 状态栏（HUD）查看当前状态：

```
[WISE] autopilot:execution | agents:3 | todos:2/5 | ctx:45%
```

| 字段 | 含义 |
|-------|---------|
| `autopilot:execution` | autopilot 流水线中的当前阶段 |
| `agents:3` | 当前活跃 agent 数量 |
| `todos:2/5` | 已完成任务 / 总任务数 |
| `ctx:45%` | 上下文窗口使用百分比 |

配置 HUD 显示：

```bash
/wise:hud setup
```

### 从小处开始

若 autopilot 感觉规模过大，可先用单任务命令：

```bash
# 代码分析
analyze why this test is failing

# 文件搜索
deepsearch for files that handle authentication

# 简单实现
ultrawork add a health check endpoint
```

这些关键词会直接调用单个合适的 agent，而不运行完整流水线。

## 配置

WISE 支持两级配置文件。

| 范围 | 文件路径 | 用途 |
|-------|-----------|---------|
| 用户（全局） | `~/.config/claude-wise/config.jsonc` | 应用于所有项目 |
| 项目 | `.claude/wise.jsonc` | 仅应用于当前项目 |

> ⚠️ **警告：** 配置文件格式为 JSONC（支持注释的 JSON）。不是 TypeScript 配置文件（`wise.config.ts`）。

### 配置优先级

当多个来源存在设置时，按以下顺序合并（越靠后优先级越高）：

```
Defaults → User config (~/.config/claude-wise/config.jsonc)
         → Project config (.claude/wise.jsonc)
         → Environment variables
```

### 基本配置结构

```jsonc
{
  // 各 agent 的模型分配
  "agents": {
    "explore": { "model": "haiku" },
    "executor": { "model": "sonnet" },
    "architect": { "model": "opus" }
  },

  // 功能开关
  "features": {
    "parallelExecution": true,
    "lspTools": true,
    "astTools": true
  },

  // 魔法关键词自定义
  "magicKeywords": {
    "ultrawork": ["ultrawork", "ulw", "uw"],
    "search": ["search", "find", "locate"],
    "analyze": ["analyze", "investigate", "examine"],
    "ultrathink": ["ultrathink", "think", "reason"]
  },

  // 可选的 prompt 级公司上下文契约
  "companyContext": {
    "tool": "mcp__vendor__get_company_context",
    "onError": "warn"
  }
}
```

### 通过 MCP 接入公司上下文

若你的组织通过自定义 MCP 服务器暴露内部指引，可在 WISE 标准配置文件中配置所选工具：

```jsonc
{
  "companyContext": {
    "tool": "mcp__vendor__get_company_context",
    "onError": "warn"
  }
}
```

- MCP 服务器本身通过常规 Claude/WISE MCP setup 流程注册。
- `tool` 为完整 MCP 工具名。
- `onError` 控制 prompt 级回退：`warn`（默认）、`silent` 或 `fail`。

这是 advisory 工作流契约，非运行时强制。完整契约见 [company-context-interface.md](./company-context-interface.md)。

### 覆盖 agent 模型

可更改各 agent 使用的 AI 模型：

```jsonc
{
  "agents": {
    // 将 explore agent 升级到更强模型
    "explore": { "model": "sonnet" },

    // 复杂项目将 executor 升级到 opus
    "executor": { "model": "opus" },

    // 节省成本：文档写作使用 haiku
    "writer": { "model": "haiku" }
  }
}
```

#### 默认模型映射

| Agent | 默认模型 | 角色 |
|-------|--------------|------|
| `explore` | haiku | 代码库探索 |
| `writer` | haiku | 文档写作 |
| `executor` | sonnet | 代码实现 |
| `debugger` | sonnet | 调试 |
| `designer` | sonnet | UI/UX 设计 |
| `verifier` | sonnet | 验证 |
| `tracer` | sonnet | 证据驱动的因果追踪 |
| `security-reviewer` | sonnet | 安全漏洞与信任边界 |
| `test-engineer` | sonnet | 测试策略与覆盖 |
| `qa-tester` | sonnet | 交互式 CLI/服务运行时验证 |
| `scientist` | sonnet | 数据与统计分析 |
| `git-master` | sonnet | Git 操作与历史管理 |
| `document-specialist` | sonnet | 外部文档与 API 参考检索 |
| `architect` | opus | 系统设计 |
| `planner` | opus | 战略规划 |
| `critic` | opus | 计划审查 |
| `analyst` | opus | 需求分析 |
| `code-reviewer` | opus | 全面代码审查 |
| `code-simplifier` | opus | 代码清晰度与简化 |

### 自定义魔法关键词

可通过 `config.jsonc` 的 `magicKeywords` 节更改四类关键词：

```jsonc
{
  "magicKeywords": {
    // 触发并行执行模式
    "ultrawork": ["ultrawork", "ulw", "parallel"],

    // 触发代码库搜索模式
    "search": ["search", "find", "locate", "grep"],

    // 触发分析模式
    "analyze": ["analyze", "debug", "investigate"],

    // 触发深度推理模式
    "ultrathink": ["ultrathink", "think", "reason"]
  }
}
```

> ℹ️ **说明：** `config.jsonc` 中的 `magicKeywords` 仅允许自定义四类：`ultrawork`、`search`、`analyze`、`ultrathink`。`autopilot`、`ralph`、`ccg` 等关键词硬编码在 keyword-detector hook 中，无法通过配置文件更改。

### 模型路由配置

WISE 根据任务复杂度自动选择模型层级：

```jsonc
{
  "routing": {
    "enabled": true,
    "defaultTier": "MEDIUM",
    // 强制所有 agent 继承父级模型
    // （使用 CC Switch、Bedrock 或 Vertex AI 时自动激活）
    "forceInherit": false
  }
}
```

| 层级 | 模型 | 适用场景 |
|------|-------|----------|
| LOW | haiku | 快速查询、简单任务 |
| MEDIUM | sonnet | 标准实现、一般任务 |
| HIGH | opus | 架构、深度分析 |

### CLAUDE.md 配置

WISE 的默认行为也通过 `CLAUDE.md` 文件配置。运行 `/wise:wise-setup` 会自动生成该文件。

| 范围 | 文件 | 说明 |
|-------|------|-------------|
| 全局 | `~/.claude/CLAUDE.md` | 跨项目共享设置 |
| 项目 | `.claude/CLAUDE.md` | 项目级上下文与覆盖 |

### 何时重新运行 setup

- 初次安装后
- WISE 更新后（以应用最新配置）
- 换用另一台机器时
- 开始新项目时（使用 `--local` 选项）
