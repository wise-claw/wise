# 本地插件安装

如何将本地开发目录中的 wise 安装为 Claude Code 插件。

## 何时使用本指南

在**本地开发 checkout 或 git worktree** 中，若希望 Claude Code 从你当前正在编辑的分支/worktree 加载插件，请使用本文档。

- **Marketplace / 插件用户**：优先使用 README 中的快速入门流程
- **npm 用户**：优先使用 `npm i -g wise@latest`；npm 安装会同时提供 `wise` 命令别名
- **本地开发 / worktree 用户**：使用本指南，使已安装插件与正在编辑的分支/worktree 保持一致

## 快速安装

```bash
# 1. 将本地目录添加为 marketplace
claude plugin marketplace add /path/to/wise

# 2. 从本地 marketplace 安装插件
claude plugin install wise@wise

# 3. 在 Claude Code 内重新运行 setup，使 CLAUDE.md / skills 反映当前 checkout
/setup

# 4. 重启 Claude Code 以加载插件
```

## 命令参考

```bash
# 列出已配置的 marketplace
claude plugin marketplace list

# 更新 marketplace（从源重新读取）
claude plugin marketplace update wise

# 更新已安装的插件
claude plugin update wise@wise

# 列出已安装的插件
claude plugin list

# 卸载
claude plugin uninstall wise@wise

# 移除 marketplace
claude plugin marketplace remove wise
```

## 插件结构

插件需要 `plugin.json` 清单文件：

```json
{
  "name": "wise",
  "version": "1.0.0",
  "description": "Multi-agent orchestration system for Claude Code",
  "hooks": {
    "PreToolUse": ["scripts/pre-tool-enforcer.mjs"],
    "PostToolUse": ["scripts/post-tool-verifier.mjs"],
    "SessionStart": ["scripts/session-start.mjs"]
  },
  "agents": ["agents/*.md"],
  "commands": ["commands/**/*.md"],
  "skills": ["skills/*.md"]
}
```

## 开发工作流

> **热重载说明**：`claude plugin marketplace add <local-folder>` 会将插件内容复制/缓存到 `~/.claude/plugins/cache/`，**不会**监视你的 checkout。每次修改 agents、skills 或 commands 后，都需要按下方步骤执行 `marketplace update` + `plugin update` + 重新运行 setup。若希望无需刷新 marketplace 即可加载变更，请改用 [替代方案](#alternative---plugin-dir-no-marketplace) 中的 `--plugin-dir` 流程。

在修改插件后（包括来自已链接 git worktree 的修改）：

```bash
# 1. 构建（若有 TypeScript 变更）
npm run build

# 2. 更新 marketplace 缓存
claude plugin marketplace update wise

# 3. 更新已安装的插件
claude plugin update wise@wise

# 4. 在 Claude Code 中重新运行 setup，使 prompts/skills 与刷新后的插件一致
/setup

# 5. 重启 Claude Code 会话
```

## 与 npm 全局安装对比

| 方式 | 命令 | 文件位置 |
|------|------|----------|
| 插件 | `claude plugin install` | `~/.claude/plugins/cache/` |
| npm 全局 | `npm install -g` | `~/.claude/agents/`、`~/.claude/commands/` |

**更推荐使用插件模式**——文件相互隔离，并使用 Claude Code 原生插件系统，通过 `${CLAUDE_PLUGIN_ROOT}` 变量解析路径。

<a id="alternative---plugin-dir-no-marketplace"></a>

## 替代方案：`--plugin-dir`（无需 marketplace）

若不想使用 marketplace 系统，可直接用 `--plugin-dir` 启动 Claude Code：

```bash
export WISE_PLUGIN_ROOT=/path/to/wise
claude --plugin-dir /path/to/wise
wise setup --plugin-dir-mode
```

或使用 npm CLI 封装（`wise`，或你偏好的长别名 `wise`），它会自动处理 `--plugin-dir`：

```bash
wise --plugin-dir /path/to/wise setup --plugin-dir-mode
# 等价的长别名：
wise --plugin-dir /path/to/wise setup --plugin-dir-mode
```

**与 marketplace 方式的主要区别：**

- 插件直接从文件系统加载（无缓存）
- 修改 agent/skill 文件后，重新运行 `wise setup` 即可生效
- 无需 marketplace update 步骤——只需重新构建并重新运行 setup
- 若直接使用 `claude`，需手动导出 `WISE_PLUGIN_ROOT`（`wise` / `wise` 封装会自动设置）

完整的决策矩阵与权威的 plugin-dir 文档，见 [REFERENCE.md 中的 Plugin directory flags 章节](./REFERENCE.md#plugin-directory-flags)。

## 故障排查

**插件未加载：**

- 安装后重启 Claude Code
- 检查 `claude plugin list` 中状态是否为 "enabled"
- 确认 `plugin.json` 存在且为合法 JSON

**仍显示旧版本：**

- 缓存目录名可能仍显示旧版本号，但实际代码来自最新 commit
- 运行 `claude plugin marketplace update`，再运行 `claude plugin update`

**使用 `--plugin-dir` 或 `--plugin-dir-mode`？**

- 确认 `WISE_PLUGIN_ROOT` 已设置：`echo $WISE_PLUGIN_ROOT`
- 若直接使用 `claude --plugin-dir`（而非 `wise --plugin-dir`），需手动导出 `WISE_PLUGIN_ROOT`
- 运行 `wise doctor --plugin-dir /path/to/wise`（或 `wise doctor --plugin-dir /path/to/wise`）进行诊断
